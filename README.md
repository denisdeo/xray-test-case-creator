# Xray Test Case Creator ‚Äî Xray-Tool

Creates Test Cases inside an Xray Test Set from a CSV file.
Works with **Xray for Jira Cloud**.

---

## How it works

The script creates **Manual Test Cases** directly inside your Xray Test Set (e.g. XRR-NUM) using the Xray Cloud GraphQL API. Each test case is created in the **X-Ray Repository (XRR)** project with its steps, then automatically linked to the Test Set and Test Plan you specify in the CSV.

---

## Setup (5 minutes)

### 1. Install Node.js
Download from https://nodejs.org ‚Äî any version 14+ works.
Verify: `node --version`

### 2. Configure your connection
```bash
cp config.example.json config.json
```
Open `config.json` and fill in all fields:
```json
{
  "jiraUrl":          "https://yourcompany.atlassian.net",
  "projectKey":       "XRR",
  "email":            "your.email@company.com",
  "apiToken":         "your-jira-api-token-here",
  "issueTypeName":    "Test Case",
  "xrayClientId":     "your-xray-client-id-here",
  "xrayClientSecret": "your-xray-client-secret-here"
}
```

> **`projectKey` must be `XRR`** ‚Äî the X-Ray Repository project where Xray stores all test cases, test sets and test plans. This is NOT your development project key (e.g. PROJ).

**Important:** `config.json` is gitignored ‚Äî never commit credentials.

---

### 3. Get your Jira API token
‚Üí https://id.atlassian.com/manage-profile/security/api-tokens
‚Üí Click **Create API token** ‚Üí copy the value ‚Üí paste into `apiToken`

---

### 4. Get your Xray Client ID and Client Secret
The script uses the **Xray Cloud GraphQL API** to create test cases and steps, which requires its own credentials separate from your Jira API token.

1. Go to your Jira ‚Üí **Apps** (top navigation)
2. In the left sidebar find **Xray** ‚Üí **Settings** ‚Üí **API Keys**
3. Click **Generate** (or **Create API Key**)
4. Give it a name (e.g. `xray-tool`)
5. Copy the **Client ID** and **Client Secret** ‚Äî the secret is only shown once
6. Paste both into `config.json`

### `--list-testset`

Lists all test cases inside an existing Xray Test Set. No input file required.

```bash
node create-test-cases.js --list-testset XRR-NUM
```

For each test case it shows:
- **Key** ‚Äî the XRR issue key (e.g. XRR-1234)
- **Ticket** ‚Äî the story or ticket the test belongs to (e.g. PROJ-100)
- **Summary** ‚Äî the test case title
- **Description** ‚Äî a short preview of the test case description

## CSV format

Open your CSV file in Excel or any spreadsheet app to edit.

**Header row (required):**
```
summary, ticket, priority, label, testSet, testPlan, preconditions, description, step_action, step_data, step_result
```

**One row per step.** The first row of each test case fills in all fields. Continuation rows (extra steps for the same test case) leave `summary` blank ‚Äî only `step_action`, `step_data`, and `step_result` are needed.

**Example:**

| summary | ticket | priority | label | testSet | testPlan | preconditions | description | step_action | step_data | step_result |
|---|---|---|---|---|---|---|---|---|---|---|
| Verify login | PROJ-100 | High | auth | XRR-NUM | PROJ-101 | User is logged out | | Navigate to the login page | | Login page is displayed |
| | | | | | | | | Enter valid credentials and submit | username / password | User is logged in successfully |
| | | | | | | | | Check the page after login | | User is redirected to the dashboard |

**Fields:**

| Column | Required | Notes |
|---|---|---|
| summary | Yes (first row only) | Test case title ‚Äî signals start of a new test case |
| step_action | Yes (each step row) | What the tester does |
| testSet | **Strongly recommended** | Xray Test Set key (e.g. XRR-NUM) ‚Äî the test case will appear inside this Test Set |
| testPlan | **Strongly recommended** | Xray Test Plan key (e.g. PROJ-101) ‚Äî links the test case to a Test Plan for execution tracking |
| step_data | No | Input values or test data |
| step_result | No | Expected outcome |
| ticket | No | Dev ticket reference ‚Äî added as a Jira label (e.g. PROJ-100) |
| priority | No | High / Medium / Low ‚Äî defaults to Medium |
| label | No | Additional Jira label |
| preconditions | No | Appears in test case description |
| description | No | Extra context in test case description |

> ‚öÝÔ∏è **testSet and testPlan are strongly recommended.** Without them, test cases are created in the XRR project but not linked to any Test Set or Test Plan ‚Äî they will be hard to find and won't appear in your Xray execution workflows.

## JSON format

Use a JSON file when test cases are generated by Claude AI or another tool. The format is an array of test case objects:

```json
[
  {
    "summary":       "Valid request to the external API returns a success response",
    "ticket":        "PROJ-100",
    "priority":      "High",
    "label":         "api-integration",
    "testSet":       "XRR-NUM",
    "testPlan":      "PROJ-101",
    "preconditions": "The application is running. The external API is accessible. A valid authentication token is configured.",
    "description":   "Optional extra context",
    "steps": [
      {
        "action": "Send a valid request to the external API.",
        "data":   "API: POST /api/orders",
        "result": "The API returns a success response."
      },
      {
        "action": "Check that the record has been saved in the database.",
        "data":   "Table: orders",
```

**JSON fields:**

| Field | Required | Notes |
|---|---|---|

| testSet | **Strongly recommended** | Xray Test Set key (e.g. XRR-NUM) |
| testPlan | **Strongly recommended** | Xray Test Plan key (e.g. PROJ-101) |

## What the script does

1. Reads the CSV and groups rows into test cases
2. Authenticates with Xray Cloud (Client ID + Secret ‚Üí Bearer token)
3. Shows a preview table ‚Äî asks for confirmation
4. For each test case, makes a **single Xray GraphQL call** (`createTest` mutation) that:
   - Creates the Test Case in the XRR project as a **Manual** test
   - Adds all steps (action / data / result) in one go
5. Links the test case to its **Test Set** (`addTestsToTestSet` mutation)
6. Links the test case to its **Test Plan** (`addTestsToTestPlan` mutation)
7. Saves results to `results.json`
8. Prints a creation log showing each XRR key created

## Troubleshooting

**"xrayClientId and xrayClientSecret are required"** ‚Äî Add both fields to `config.json`. See Setup step 4 above.

**"HTTP 401"** ‚Äî Jira API token or email is wrong. Re-check `config.json`.

**"HTTP 401 on Xray"** ‚Äî Client ID or Client Secret is wrong. Regenerate from Jira ‚Üí Apps ‚Üí Xray ‚Üí API Keys.

**"cannot create a test step because test is not of a type with kind steps"** ‚Äî The test case was created in the wrong Jira project. Make sure `projectKey` in `config.json` is `XRR`, not your dev project (e.g. PROJ).

**"Specify a valid issue type"** ‚Äî The `issueTypeName` in `config.json` doesn't match your project. To find the correct name run:
```bash
node -e "const https=require('https'),cfg=require('./config.json'),auth='Basic '+Buffer.from(cfg.email+':'+cfg.apiToken).toString('base64');https.get({hostname:new URL(cfg.jiraUrl).hostname,port:443,path:'/rest/api/2/project/'+cfg.projectKey,headers:{Authorization:auth,Accept:'application/json'}},r=>{let d='';r.on('data',c=>d+=c);r.on('end',()=>JSON.parse(d).issueTypes.forEach(t=>console.log(t.name)))})"
```

**"HTTP 403"** ‚Äî Your Jira user may not have permission to create issues or link test sets/plans in the XRR project.

**Rate limit errors** ‚Äî The script adds a 400 ms delay between requests. If you still hit limits, increase the delay in the script.
